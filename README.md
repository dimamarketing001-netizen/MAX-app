# Мини-приложение "Личный кабинет юриста" для MAX Messenger

Это стартовый шаблон для создания мини-приложения, которое будет работать внутри мессенджера MAX. Приложение представляет собой личный кабинет для клиентов юридической компании.

## Архитектура

Проект состоит из трех основных частей:

1.  **Фронтенд (этот репозиторий):** React-приложение, использующее `MAX UI` для интерфейса и `MAX Bridge` для взаимодействия с клиентом MAX. Отвечает за отображение данных и взаимодействие с пользователем.
2.  **Бэкенд (нужно будет создать):** Серверное приложение (например, на Node.js, Python), которое выполняет всю бизнес-логику:
    *   **Валидация:** Проверяет подлинность данных пользователя, полученных от MAX, используя ваш `BOT_TOKEN`.
    *   **Авторизация:** Находит пользователя в вашей базе данных по `user.id` из MAX.
    *   **Интеграция с Bitrix24:** По запросу от фронтенда обращается к API или вебхукам Bitrix24 для получения актуальных данных о сделках, счетах и статусах.
    *   **API для фронтенда:** Предоставляет эндпоинты, по которым фронтенд может запросить данные пользователя.
3.  **Платформа MAX:** Среда, в которой исполняется мини-приложение. Предоставляет данные о пользователе и API для взаимодействия с мессенджером.

## Пошаговый план запуска

### 1. Настройка на платформе MAX

- **Создайте бота:** Зайдите на [платформу MAX для партнёров](https://max.ru/partners), создайте организацию и чат-бота. Сохраните `BOT_TOKEN` — он понадобится для валидации данных на бэкенде.
- **Подготовьте хостинг:** Соберите и загрузите файлы этого фронтенд-проекта на любой статический хостинг (Vercel, Netlify, GitHub Pages и т.д.). Вам нужен публичный URL вида `https://your-app-url.com`.
- **Свяжите бота и мини-приложение:** В настройках вашего бота на платформе MAX перейдите в раздел "Чат-бот и мини-приложение" и вставьте URL вашего размещенного приложения. Выберите название для кнопки (например, "Личный кабинет").

### 2. Разработка бэкенда

Вам необходимо создать сервер, который будет:

1.  Принимать запрос от фронтенда, содержащий `initData` (доступно как `window.WebApp.initData`).
2.  Выполнять **валидацию данных**, как описано в документации MAX. Это критически важно для безопасности.
3.  Используя `user.id` из валидированных данных, находить клиента в вашей CRM.
4.  Через API Bitrix24 получать его сделки, платежи и статусы.
5.  Формировать и отдавать JSON с данными для отображения в личном кабинете.

### 3. Доработка фронтенда

- В файле `src/App.jsx` найдите `useEffect`. Сейчас он использует моковые (тестовые) данные.
- Замените его на реальный `fetch` запрос к вашему бэкенду.
- Передавайте в запросе `window.WebApp.initData` для валидации на сервере.
- Обработайте полученные с бэкенда данные и отобразите их в интерфейсе.

## Запуск проекта локально

1.  Установите зависимости: `npm install`
2.  Запустите dev-сервер: `npm run dev`

Для полноценного тестирования с `MAX Bridge` приложение нужно открывать внутри мессенджера MAX.
